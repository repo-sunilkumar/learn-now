font-style: normal; font-variant: normal; font-weight: normal; line-height: normal;&quot;&gt;&lt;span style=&quot;font-family: Arial, Verdana; font-size: small;&quot;&gt;A static constructor does not take access modifiers or have parameters.&lt;/span&gt;&lt;/li&gt;&lt;li style=&quot;font-family: Arial, Verdana; font-style: normal; font-variant: normal; font-weight: normal; line-height: normal;&quot;&gt;&lt;span style=&quot;font-family: Arial, Verdana; font-size: small;&quot;&gt;A static constructor is called automatically to initialize the class before the first instance is created or any static members are referenced.&lt;/span&gt;&lt;/li&gt;&lt;li style=&quot;font-family: Arial, Verdana; font-style: normal; font-variant: normal; font-weight: normal; line-height: normal;&quot;&gt;&lt;span style=&quot;font-family: Arial, Verdana; font-size: small;&quot;&gt;A static constructor cannot be called directly.&lt;/span&gt;&lt;/li&gt;&lt;li style=&quot;font-family: Arial, Verdana; font-style: normal; font-variant: normal; font-weight: normal; line-height: normal;&quot;&gt;&lt;span style=&quot;font-family: Arial, Verdana; font-size: small;&quot;&gt;The user has no control on when the static constructor is executed in the program.&lt;/span&gt;&lt;/li&gt;&lt;li style=&quot;font-family: Arial, Verdana; font-style: normal; font-variant: normal; font-weight: normal; line-height: normal;&quot;&gt;&lt;span style=&quot;font-size: small;&quot;&gt;It can only access the static member(s) of the class.Because&amp;nbsp;Non static member is specific to the object instance. If static constructor are allowed to work on non static members it will reflect the changes in all the object instance, which is impractical.&lt;/span&gt;&lt;/li&gt;&lt;li style=&quot;font-family: Arial, Verdana; font-style: normal; font-variant: normal; font-weight: normal; line-height: normal;&quot;&gt;&lt;span style=&quot;font-size: small;&quot;&gt;There should be no parameter(s) in static constructor.&amp;nbsp;Since, It is going to be called by CLR, nobody can pass the parameter to it.&amp;nbsp;&lt;/span&gt;&lt;/li&gt;&lt;li style=&quot;font-family: Arial, Verdana; font-style: normal; font-variant: normal; font-weight: normal; line-height: normal;&quot;&gt;&lt;span style=&quot;font-size: small;&quot;&gt;Only one static constructor is allowed. Because&amp;nbsp;Overloading needs the two methods to be different in terms of method/constructor definition which is not possible in static constructor.&lt;/span&gt;&lt;/li&gt;&lt;li style=&quot;font-family: Arial, Verdana; font-style: normal; font-variant: normal; font-weight: normal; line-height: normal;&quot;&gt;&lt;span style=&quot;font-size: small;&quot;&gt;There should be no access modifier to it.Because&amp;nbsp;Again the reason is same call to static constructor is made by CLR and not by the object, no need to have access modifier to it.&lt;/span&gt;&lt;/li&gt;&lt;li&gt;&lt;font face=&quot;Arial, Verdana&quot; style=&quot;font-size: small;&quot;&gt;It is thread-safe. Suppose several threads want to get the property Instance at once. One of the threads will be told to first run the type initializer (also known as the static constructor). Meanwhile all other threads wanting to read the Instance property, will be locked until the type initializer has finished. Only after the field initializer has concluded, will threads be allowed to get the Instance value. So no-one can see Instance being null&lt;/font&gt;&lt;/li&gt;&lt;/ol&gt;&lt;/div&gt;
